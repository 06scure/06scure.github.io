![](https://qiniu.mingxuan.xin/picgo/20260302235245089.png)
直接把刚才博客的命令搬过来，都是常用的：
1. 由用户视图进入系统视图

```
system-view
```
2. 由用户视图进入系统视图

```
sysname R1
```
3. 查看路由器当前配置信息
```
display current-configuration
```
4. 查看接口信息
```
display interface
```
5. 进入接口视图
```
interface type number
```
注意，交换机必须进入系统视图才能去配置，因为系统视图的权限比较大，可作参数修改，设定系统功能，比如划分VLAN，配置路由，设定访问方法，设定速率；用户视图权限小，只能看看基本配置信息，只读的，不能修改。
## 三、实验步骤
摸索差不多了，开始继续实验内容，根据这篇[博客](https://blog.51cto.com/jackyzhang/1034893)的内容，我们要对交换机操作一下：
![](https://qiniu.mingxuan.xin/picgo/20260302235245090.png)
这里先启动，然后打开CLI（个人理解就是交换机的命令行），然后输入代码：

```
system-view
sysname R1
vlan batch 10 20
```
![](https://qiniu.mingxuan.xin/picgo/20260302235245091.png)
可以看到方括号[]是系统视图，<>尖括号是用户视图。然后继续定义接口：

```
interface GigabitEthernet 0/0/1
port link-type access
port default vlan 10
quit
```
![](https://qiniu.mingxuan.xin/picgo/20260302235245092.png)这样第一个端口就设置好了。接下来设置第二个端口
```
interface GigabitEthernet 0/0/2
port link-type access
port default vlan 20
quit
```
![](https://qiniu.mingxuan.xin/picgo/20260302235245093.png)
然后要让两个交换机连接，所以要设置trunk接口

```
interface GigabitEthernet 0/0/24
port link-type trunk
port trunk allow-pass vlan 10 20
quit
```

然后是下一台交换机，注意这些参数不是乱写的，是照着题目要求。
![](https://qiniu.mingxuan.xin/picgo/20260302235245094.png)
```
system-view
sysname R2
vlan batch 10 20
```
```
interface GigabitEthernet 0/0/1
port link-type access
port default vlan 10
quit
```
```
interface GigabitEthernet 0/0/2
port link-type access
port default vlan 20
quit
```
```
interface GigabitEthernet 0/0/24
port link-type trunk
port trunk allow-pass vlan 10 20
quit
```
![](https://qiniu.mingxuan.xin/picgo/20260302235245095.png)
## 四、验证结果
结果应该是VLAN10的互相能连上，VLAN20的互相能连上，这个时候要打开PC ping对方的地址试试
![](https://qiniu.mingxuan.xin/picgo/20260302235245096.png)

![](https://qiniu.mingxuan.xin/picgo/20260302235245097.png)
这里是用PC1去PING PC3和PC4可以看到1和3在一个VLAN网络中，可以连接上，2和4在VLAN中，所以1不能和4连接。

可以顺便打开一下wireshark看一眼抓包，我这里抓的是交换机的24端口，1到4的数据没有通过24端，而1到3的通过了。
![](https://qiniu.mingxuan.xin/picgo/20260302235245098.png)
然后尝试用两个S3700去搭建交换机，发现3700只有2个GE端口，和5700不一样，所以配置代码也应该不一样，如果按上述步骤直建议用5700。
